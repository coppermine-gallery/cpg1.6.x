Créer un thème pour Coppermine qui reprend l'aspect général de votre site

Introduction

Coppermine peut faire beaucoup de choses extraordinaires. L'une de ces choses qui peut être faite facilement avec Coppermine est de l'appeler via une commande include de PHP, sans que ce soit une option. Que pouvez vous faire alors? Certains décident de placer Coppermine dans un cadre (frame) ou dans une iframe à l'intérieur de leur site, ce qui fonctionne comme souhaité, mais qui n'est pas une bonne idée: tous les inconvénients des cadres ou des iframes s'appliquent à cette solution bien évidemment. Donc, quelle est la méthode à préférer pour que votre galerie s'intègre totalement à votre site: vous devrez crére un thème qui s'adapte complètement à l'aspect général de vos pages.

Cette section est faite pour vous explique comment faire cela.

Pré-requis

Pour être en mesure d'utiliser ce tutoriel, vous devrez remplir un certain nombre de conditions:

Outils

  • Navigateur

    A priori vous en avez un - vous ne pouriez par lire cette documentations sinon. L'équipe de développement recommande l'utilisation de Firefox associé à la barre d'outils web developer

  • Editeur

    Vous aurez besoin d'un éditeur de texte. Si vous utilisez Windows, vous pouvez utiliser notepad.exe, mais nous recommandons de meilleurs outils comme Notepad++. Les éditeurs WYSIWYG comme Frontpage ou Dreamweaver n'entrent pas dans la catégorie des "éditeurs de texte" et ne doivent pas être utilsiés pour modifier les fichiers Coppermine, parcequ'ils ont tendance à modifier le code alors qu'ils ne le devraient pas.
    Dés recommandations plus détaillées peuvent être trouvées dans la pageDetailed recommendations for editors can be found on the page Documentation du développeur → Outils recommandés par les développeurs → Editeurs de texte

  • Application FTP

    Bien entendu vous devrez être en mesure de télécharger votre thème personnalsié sur votre serveur, donc vous devrez avoir votre propre client FTP (vous en avez probablement déjà un si vous avez une page web).
    Pour une liste des clients FTP recommandés, voyez la Documentation du développeur → Outils recommandés par les développeurs → Clients FTP

Connaissances

  • Des connaissances basiques en HTML et CSS sont tout ce dont vous aurez besoin. Vous n'avez pas besoin de connaitre/de comprendre le code PHP qui se trouvera dans votre thème personnalisé
  • être prêt à passer du temps: un programmeur expérimenté pourra créer un thème personnalisé basé sur un thème existant en quelques minutes. Une adaptation précise prendra plus de temps.

Prenez d'abbord un thème similaire

La première étape est de choisir un thème qui correspond le mieux au désign de votre site. Pour cela, vous pouvez choisir un thème qui est fourni avec le pack de Coppermine ou prendre un thème proposé en contribution par un utilisateur.

Dans ce tutoriel, nous utiliserons le thème classic comme exemple de base de départ.

Comme expliqué dans la section "Renommer votre thème d'abbord", vous ne devez pas commencer à modifier directement le thème clasic, mais créer une copie de ce thème. Cette étape est essentielle pour être sécurisé dans lors des futures mises à jour. Pour ce tutoriel, nous supposons que vous avez fait comme suggéré et renommé votre copie du thème classic en "mon_theme".

Suivez les autres étapes de la même manière: lancez votre éditeur de texte, ouvrez themes/mon_theme/template.html et faites un trouver/remplacer pour la chaine "classic" (sans les guillemets) et remplacez la par la chaine "mon_theme" (toujours sans les guillemets). Sauvegardez vos changements dans un fichier temporaire nommé "template_copy.html" dans le nouveau répertoire, mais ne fermez pas l'éditeur de texte - nous en aurons encore besoin.

Commencez à modifier votre thème

Ce que nous allons faire maintenant est d'utiliser la sortie crée par votre site en général pour créer un fichier template personalisé. Pour cela, affichez votre site dans votre navigateur comme exemple - bien sur vous ferez cela avec votre vrai site plutôt qu'avec l'exemple.

En regardant la page d'index de votre site dans votre navigateur, faites un "clic droit" n'impotrte ou dans un endroit vide de la sortie et choisissez "voir la source" du menu contextuel qui s'affiche. Une fenêtre devrait apparaitre qui affiche le code HTML complet de la page: sélectionnez le (l'ensemble du contenu de la fenêtre du code source) et copiez le dans le presse papier. Puis allez dans votre éditeur de texte (par exemple lancez notepad) et collez el contenu du prese papier dans une fenêtre vide. Maintenant sauvegardez cela dans le répertoire de votre nouveau thème personnalisé (dans notre tutoriel, le nom du dossier est themes/mon_theme/). Comme nom de fichier, choisissez template.html. L'éditeur de texte devrait vous avertir que ce fichier existe déjà, ce qui est normal puisque nous avons créer une copie de ce fichier, vous pouvez donc sans risque le remplacer. Maintenant nous avons la sortie HTML de votre site général dans un fichier template.

Le travail le plus dur est fait, allons maintenant aux choses plus faciles: certains conteneurs sont obligatoires, d'autres optionnels. Jetons un oeil aux autres conteneurs obligatoires et ajouttons les au bon endroit:

  • Le conteneur {LANG_DIR}
    dans themes/mon_theme/template.html, trouvez la balise d'ouverture <html> - dans notre exemple elle ressemble à ça: 
    <html xmlns="http://www.w3.org/1999/xhtml" xml:lang="en" lang="en">
    Remplacez là par 
    <html xmlns="http://www.w3.org/1999/xhtml" xml:lang="en" lang="en" dir="{LANG_DIR}">
  • Le conteneur {CHARSET}
    Quel que soit la définition qur vous utilisez dans votre site pour le jeu de caractères, il est obligatoire de remplacer cette définition du jeu de caractères par le conteneur de Coppermine - trouvez 
    <meta http-equiv="Content-Type" content="text/html; charset=UTF-8" />
    et remplacez par 
    <meta http-equiv="Content-Type" content="text/html; charset={CHARSET}" />
    Si une telle ligne qui défini le jeu de caractères n'existe pas pour votre page générale, ajoutez cette ligne directement après la balise <head>.
  • Le conteneur {TITLE}
    Trouvez la balise d'ouverture/fermeture <title>- et remplacez le contenu entre les balises d'ouverture et de fermeture par le conteneur {TITLE}. dans notr eexemple, remplacez 
    <title>Example page</title>
    par 
    <title>{TITLE}</title>
  • Le conteneur {META}
    Coppermine utilise/complète certaines méta-balises, vous devrez donc enlever les méta-balises existantes dans le fichier template.html et les remplacer par le conteneur correspondant. Dans notre exemple, il n'y a pas de méta-balises, il n'y a donc rien à remplacer. Si c'est aussi le cas pour vous, ajoutez le conteneur 
    {META}
    dans une nouvelle ligne directement apès la balise de fermeture title (c.a.d après </title>).
    Si vous avez déjà des méta-balises dans le design général de votre site, remplacez les toutes par le conteneur {META} de Coppermine.
  • Le conteneur {JAVASCRIPT}
    Ce conteneur est souvent oublié par les débutants qui veulent créer leur propre thème - c'est une erreur commune, parceque les débutants pensent que JavaScript n'est pas important. Néanmoins, ce conteneur est remplacé, lorsque la galerie fonctionne par des scripts vitaux pour Coppermine. Ceci dit, le conteneur {JAVASCRIPT} ne remplace rien, mais est à placer directement avant la balise de fermeture </head>. Ajoutez le conteneur dans une nouvelle ligne avant 
    </head>
  • Le conteneur {MESSAGE_BLOCK}
    Le bloc de message ne sera généré que si un message important doit être affiché à l'utilisateur final. Il est directement relié à ce qui se passe et doit se trouver directement avant le conteneur {GALLERY}, vous devez donc l'ajouter dans une nouvelle ligne juste avant la ligne contenant le conteneur {GALLERY}.

Adapter le style

A présent, nous avons terminé avec les appels de conteneurs obligatoires - maintenant, regardons la feuille de style (CSS). Dans notre petut exemple, il est très facile d'intégrer le CSS qui se trouve dans le design général du site dans la feuille de style de Coppermine, puisqu'il n'y a que peu de définitions, et que pour une facilité d'utilisations, elles se trouvent dans le fichier de template (styles inclus). Alors que c'est une bonne idée pour une petite démonstration pour rendre le code lisible, il est généralement recomendé de placer les styles dans une feuille de style externe. C'est pourquoi Coppermine le fait. Dans chaque thème il y a un fichier nommé "style.css" contenant toutes les CSS. C'est le cas pour le thème que nous utilisons comme base pour notre thème personnalisé. Pour celà, ouvrez une autre fenêtre dans votre éditeur de texte et ouvrez le fichier themes/mon_theme/style.css. Laissez cette fenêtre ouverte - nous en aurons besoin pendant une bout de temps.

D'abbord, nous devons "prévenir" notre fichier HTML qu'il existe une feuille de style externe qui doit être prise en compte. Pour cela, regardez la fenêtre de l'éditeur contenant themes/mon_theme/template_copy.html: cherchez >link rel="stylesheet" href="themes/my_theme/style.css" type="text/css" />. Copiez cette ligne dans themes/mon_theme/template_copy.html, juste après l'appel au conteneur {META} . Comme vous pouvez le voir, elle ne contient pas juste une référence à style.css, mais à themes/my_theme/style.css.

La raison de ce chemin "étrange" est que vous devez avoir en tête que le thème sera par la suite interprété lors de l'execution de Coppermine - cela se passera depuis le répertoire racine de votre galerie Coppermine. C'est la raison pour laquelle le chemin vers toutes les ressources soit être vu depuis le répertoire racine de votre installation Coppermine. Vous noterez par la suite que ce n'est pas uniquement le cas pour la feuille de style, mais aussi pour d'autres ressources similaires (comme les images inclues dans votre thème).

Nous avons maintenant demandé à notre thème personnalisé de prendre en compte la feuille de style - ce qui reste à faire maintenant est de déplacer les CSS inclus dans le fichier template.html dans notre feuille de style externe.
